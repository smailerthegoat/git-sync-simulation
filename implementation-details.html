Generates random dummy data commits on a branch.

  • Total size per commit: 0.8 – 3.2 MB (random)
  • Files per commit: 1 – 4 (random split)
  • File types: .bin, .dat (random bytes), .txt (repeated text)
  • Directory structure: data/{group_name}/{ISO_timestamp}/
  • Commit message: Includes total_size_mb field used by sync engine for size parsing
  • Key function: generate_commit(repo, branch_name, group_name, rng) → (hexsha, file_meta)

simulation/simulation/group_worker.py 78 lines

Per-branch daemon thread that generates commits at random intervals.

  • Interval: 12 – 22 seconds between commits (configurable via rng.uniform)
  • Per commit: Acquires shared lock → git checkout branch → generate_commit() → posts event to queue
  • Push support: Optional enable_push flag pushes after each commit
  • Clean shutdown: stop() sets a threading.Event; worker checks every 0.3s
  • Lock injection: _lock attribute set by main.py for checkout serialization

simulation/simulation/sync_engine.py 119 lines

Core sync logic: extracts patches from source, filters via policy, applies to destination.

  • Patch extraction: git format-patch generates .patch files in a temp directory
  • Size parsing: Regex extracts total_size_mb from patch content
  • Policy filtering: Calls destination branch's policy.filter_patches()
  • Patch application: git am --ignore-whitespace applies accepted patches
  • Merge commit: Empty commit recording policy decisions (accepted/rejected MB, quota state)
  • State tracking: _last_synced dict remembers last synced SHA per (src, dst) pair
  • Error handling: git am --abort on failure, continues with remaining patches

simulation/simulation/cycle_manager.py 57 lines

Manages 60-second quota cycles and triggers recalculation.

  • Cycle length: 60 seconds (configurable)
  • End-of-cycle: Increments cycle_number, calls recalculate_quota() on all StoragePolicy instances
  • Carry-over formula: new_quota = remaining_bytes / cycle_length_seconds
  • Event: Posts cycle_end with quota update details (old_mb, new_mb per group)

simulation/simulation/sync_scheduler.py 59 lines

Triggers SyncEngine.sync() at random intervals with manual override support.

  • Interval: 25 – 45 seconds between automatic syncs
  • Branch selection: rng.sample(branches, k=2) picks random (src, dst) pair
  • Manual trigger: trigger_manual(src, dst) queues immediate sync (polled every 0.3s)
  • Error handling: Posts error event on sync failure, does not crash the thread

4. Policy Mechanics

StoragePolicy (group-storage)

Constraint model: Dual-limited by per-sync bandwidth and per-cycle quota.

sync_budget = min(bandwidth_limit, quota_remaining)

For each patch in order:
    if accumulated + patch.size_bytes <= sync_budget:
        ACCEPT patch
        accumulated += patch.size_bytes
    else:
        REJECT patch

quota_remaining -= accumulated   (tracked via accepted_this_cycle)

Carry-Over Logic (end of cycle)

remaining = quota_bytes - accepted_this_cycle
new_quota = remaining // cycle_length_seconds
accepted_this_cycle = 0

This creates a "starvation" effect: if a group doesn't use its full quota, the carry-over divides the remainder by 60, yielding a tiny next-cycle budget. This incentivizes efficient bandwidth usage.

PrivacyPolicy (group-privacy)

STUB Currently a no-op — accepts all patches unconditionally.

Prepared for future implementation with redact_emails and exclude_patterns config hooks, plus an apply_privacy_filter() method.

UnconstrainedPolicy (group-unconstrained)

PASS-THROUGH Accepts all patches with no restrictions. Returns total size in reason string.

5. Sync Engine Detail

Step-by-Step Sync Process

  1. Check last synced: Look up _last_synced[(src, dst)] for the SHA of the last synced commit
  2. Generate patches: git format-patch --output-directory /tmp/gitsync_xxx {since}..{src}
  3. Parse sizes: Regex total_size_mb:\s*([\d.]+) extracts MB from patch content; defaults to 0.5 MB if missing
  4. Policy filter: Call policies[dst].filter_patches(patches, cycle_number)
  5. Apply accepted: git checkout dst then git am --ignore-whitespace for each accepted patch
  6. Merge commit: Empty commit on dst recording: policy name, accepted/rejected MB, patch counts, quota state
  7. Update tracking: Store current src HEAD as _last_synced[(src, dst)]
  8. Post event: SYNC event with all metrics to the shared queue

7. Threading & Concurrency Model

ThreadDaemon?Shared Resources
GroupWorker ×3Yesgit.Repo, Lock, Queue
CycleManagerYesPolicies dict, Queue
SyncSchedulerYesSyncEngine, Queue

Lock strategy: A single threading.Lock serializes all git checkout operations. Workers acquire the lock before checkout + commit, and the SyncEngine also checks out branches under the same lock (implicitly, via the repo object). This prevents concurrent checkouts from corrupting the git index.

Event queue: All threads post structured dicts to a shared queue.Queue. The main loop pulls events with a 1-second timeout and logs them. This decouples event production from consumption.

10. Usage

# First-time setup
bash bootstrap_inside_repo.sh

# Activate virtual environment
source .venv/bin/activate

# Run simulation (headless, 60 seconds)
python simulation/main.py --duration 60 --seed 42

# Run with remote push enabled
python simulation/main.py --duration 180 --seed 42 --push

# Clean between runs (reset branches to pre-data state)
python clean.py

# Launch Streamlit dashboard (skeleton)
streamlit run simulation/app.py
